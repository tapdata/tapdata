# DuckDB HA方案 - 备份文件方案

## 现状

- 实现：io/tapdata/flow/engine/V2/node/hazelcast/processor/HazelcastDuckDbSqlNode.java
- 已支持单机模式下的DuckDB本地计算
- 不支持HA高可用，服务发生故障导致任务迁移到其他节点时会丢失DuckDB数据导致任务无法恢复

## 场景

- 任务上线运行
- 引擎离线任务重新调度
- 引擎未离线手动任务调度

## 大纲

- 任务运行以及维护本地DuckDB数据和现有的单机模式保持一致
- 任务运行开始时启动一个并行线程并定期同步DuckDB数据文件到中间库，以Grid文件为参考目标
- 所有的Grid文件都要与任务进行关联关系的建立，利用中间库来存储，利于任务切换引擎后缓存恢复
- 并行上传DuckDB数据文件的线程需要能够保证每次上传的文件都能对应到正确的增量Offset，下次任务发生故障转移恢复时从这个增量Offset恢复，保证数据不丢
- Grid文件需要按任务分块，具体的分块原则：任务/节点/表/增量offset，任务发生故障转移恢复时也利用这个结构快速下载文件到本地目录

## 设计目标

- DuckDB高可用
- 数据一致性，保证不能丢数
- HA模式下，性能不能比单机模式低太多
