# AffectedKeyCalculator 极端场景测试设计

## 概述
为 `AffectedKeyCalculator` 增加全面的极端场景测试用例，特别聚焦于 ABA 场景的 3 维度（主键重复、操作类型、Join key 变化）测试。

## 现有测试覆盖分析

### 已覆盖（无需重复）
- 主表事件（Insert/Update/Delete）
- 从表查询（含自定义 JOIN）
- 未知表处理
- 缺失主键
- 空事件/空结果
- 查询失败异常
- 多从表配置
- 空主键值
- 数字/字符串主键类型
- 多主键处理
- 空配置处理
- after/before 字段提取

## 新增测试用例

### 1. Join key 为非主键字段
**场景：** 关联字段不是主键（如 email、user_name）
**验证：** 能正确提取和查询非主键字段

### 2. 多表链式关联
**场景：** users → orders → order_items（三层关联）
**验证：** 链式查询能正确追溯到主表主键

### 3. 主键类型混合
**场景：** 同一查询返回数字和字符串混合的主键
**验证：** 类型转换和去重正确性

### 4. 批量处理边界
**场景：** 999、1000、1001 条数据
**验证：** 批次分割和合并逻辑正确

### 5. 空值/空字符串处理
**场景：** 事件中包含空值或空字符串
**验证：** 正确过滤无效值

### 6. SQL 特殊字符处理
**场景：** 主键包含单引号、反斜杠等
**验证：** 转义逻辑正确

### 7. 大小写敏感测试
**场景：** 表名大小写不同（"orders" vs "ORDERS"）
**验证：** 忽略大小写匹配正确

### 8. 重复关联结果去重
**场景：** 不同源键关联到同一个主键
**验证：** 结果集去重正确

### 9. 查询返回部分 null
**场景：** 查询结果中某些行主键为 null
**验证：** 正确过滤 null 值

### 10. 同一子表多 Join Key 关联
**场景1（单查询多条件）：** 同一个JOIN查询使用多个关联条件
- 例如：`SELECT u.id FROM users u JOIN orders o ON u.id = o.user_id OR u.id = o.customer_id WHERE o.order_id IN (${pkValues})`
- 验证：一个事件通过多个关联路径都能找到相关主键

**场景2（多配置切换）：** 测试时动态切换不同的join key配置
- 验证：不同配置互不干扰，各自正确工作

## ABA 场景重点测试（3 维度）

### 维度说明
1. **主键重复模式** - 主键值出现的规律
2. **操作类型** - CDC 事件的操作类型（增/删/改）
3. **Join key** - 关联字段的变化

## 预期行为规范

### 通用行为
- 相同主键在结果集中只出现一次（去重）
- 保留首次出现的顺序（LinkedHashSet）
- 操作类型不影响去重逻辑
- Join key 变化可能导致关联到不同主键

### 批次处理
- 超过 1000 条数据自动分批
- 批次间去重正确
- 批次顺序不影响最终结果
