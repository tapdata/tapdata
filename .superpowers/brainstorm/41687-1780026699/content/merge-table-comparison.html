⚙️ 三、MergeTableNode 关键技术点提取

🔧 技术点 1：Formily Schema 规范

// ✅ 标准结构
{
  type: 'object',
  properties: {
    fieldName: {
      type: 'string',                    // 数据类型
      title: 'i18n_key',                 // 显示名称（国际化）
      default: 'defaultValue',           // 默认值
      required: true/false,             // 是否必填
      
      // 装饰器（布局、验证、样式）
      'x-decorator': 'FormItem',
      'x-decorator-props': {
        tooltip: '提示文本',            // 悬浮提示
        layout: 'horizontal',           // 布局方式
      },
      
      // 组件类型
      'x-component': 'Input' | 'Select' | 'Switch' | ...,
      'x-component-props': {
        // 组件属性
      },
      
      // 条件显示
      'x-visible': '{{condition}}',
      
      // 动态逻辑
      'x-reactions': [
        '{{useAsyncDataSource(...)}}',
        { fulfill: { state: {...} } }
      ],
    }
  }
}

💡 应用到 DuckDbSqlNode：

  • ✅ 所有参数必须使用 x-decorator 包装
  • ✅ 所有 title 必须使用 i18n.t() 国际化
  • ✅ 必须提供 default 默认值
  • ✅ 必填字段标记 required: true

🌳 技术点 2：复杂嵌套结构（mergeProperties 示例）

// ⭐ Array 类型 + items 对象（支持动态增删）
mergeProperties: {
  type: 'array',                        // 数组类型
  required: true,
  'x-component': 'MergeTableTree',      // 自定义树形组件
  
  items: {                              // 每个子项的结构
    type: 'object',
    properties: {
      id: { type: 'string', 'x-display': 'hidden' },
      
      // 子项内容容器
      itemsWrap: {
        type: 'void',
        'x-component': 'FormContent',
        properties: {
          mergeType: { /* Radio.Group */ },
          
          // 更深层嵌套
          wrap: {
            type: 'void',
            'x-component': 'FormContent',
            properties: {
              targetPath: { /* Input */ },
              
              // 内部数组（关联条件）
              joinKeys: {
                type: 'array',
                'x-component': 'ArrayTable',   // 表格组件
                items: { /* 列定义 */ },
                properties: {
                  addition: { /* 添加按钮 */ }
                }
              }
            }
          }
        }
      }
    }
  }
}

💡 应用到 fromTables 参数：

  • ✅ 使用相同的 Array + items 结构
  • ✅ 每个 fromTable 包含：tableName, primaryKey, joinConditions
  • ✅ 可考虑复用或改造 MergeTableTree 组件
  • ✅ 或开发轻量级的 FromTablesList 组件

🔄 技术点 3：条件显示与联动逻辑

// ✅ 方式 1: x-visible（完全隐藏/显示）
externalStorageId: {
  'x-visible': '{{$isDaas}}',          // 全局变量判断
}

// ✅ 方式 2: x-reactions（动态响应）
{
  'x-reactions': [
    // 异步数据加载
    '{{useAsyncDataSourceByConfig({service: loadExternalStorage})}}',
    
    // 状态联动
    {
      fulfill: {
        state: {
          value: '{{$self.value || $self.dataSource?.find(...)?.value}}'
        }
      }
    },
    
    // 输入验证
    {
      effects: ['onFieldInputValueChange'],
      fulfill: {
        run: `{{ 
          const arr = $self.value.split('.')
          if (arr.length > 3) {
            $self.value = arr.slice(0,3).join('.')
            $self.description = '最多支持3层'
          } 
        }}`
      }
    }
  ]
}

// ✅ 方式 3: Switch 控制（推荐用于 DuckLake/物化视图）
duckLakeEnabled: {
  type: 'boolean',
  'x-component': 'Switch',
  'x-component-props': {
    onChange: '{{(val) => toggleDuckLakeConfig(val)}}'
  }
},
duckLakeStorageType: {
  'x-visible': '{{$form.values.duckLakeEnabled === true}}'
}

💡 DuckDbSqlNode 应用场景：

  1. duckLakeEnabled → 控制 DuckLake 其他 2 个参数显示
  2. wideTablePrimaryKey → 控制物化视图其他参数显示
  3. batchSize → 根据 memoryLimitMB 自动计算最大值
  4. querySql → 实时语法校验（SQL 格式检查）

🧩 技术点 4：自定义组件规范

MergeTableNode 使用的自定义组件：

组件名 用途 Props
MergeTableTree 主从关系树形配置 treeWidth, findNodeById, loadFieldsMethod
MergeTableCache 缓存策略配置 value, findNodeById
SchemaPreview 输出 Schema 预览 无特殊 props
ClipboardButton 复制节点 ID 按钮 content, tooltip, finishTooltip

🎯 DuckDbSqlNode 需要的自定义组件：

  • DuckDbSqlConfig ✅ 已有 - SQL 编辑器（带语法高亮、自动补全）
  • FromTablesConfig 🔨 待开发 - 从表列表配置（类似简化版 MergeTableTree）
  • DuckDbPerformanceMonitor 🔨 可选 - 性能监控面板（实时显示内存/CPU）

📋 四、当前 DuckDbSql.js vs MergeTable.js - 差异清单
