🗄️ DuckLake 持久化存储（3个参数）

DuckLake 是什么？
DuckDB 默认使用内存数据库（进程重启后数据丢失）。DuckLake 提供了持久化能力，将数据存储到磁盘，支持跨会话保留。

1️⃣ duckLakeEnabled

类型: Boolean
默认值: false
@EqField: ✅

功能描述：

  • ✅ 启用/禁用 DuckLake 持久化存储
  • 🔄 当设为 true 时，数据写入磁盘而非仅内存
  • ⚡ 影响性能：磁盘 I/O vs 纯内存
💡 使用场景：
• 大数据量任务（超过内存容量）
• 需要断点续传的任务
• 生产环境长期运行的任务

2️⃣ duckLakeStorageType

类型: String
默认值: "LOCAL"
可选值: "LOCAL" | "S3" | "GCS" | "AZURE"
@EqField: ✅

功能描述：

  • 📁 决定数据的存储位置和方式
  • 🌐 支持云存储集成（S3/GCS/Azure）
  • 💻 LOCAL: 本地文件系统
💡 使用场景：
• 开发环境：LOCAL（快速调试）
• 生产环境：S3/GCS（可扩展、高可用）
• 混合云：AZURE（Azure 云用户）

3️⃣ duckLakeStoragePath

类型: String
默认值: "/tmp/ducklake"
@EqField: ✅

功能描述：

  • 📍 数据文件的存储路径
  • 📂 LOCAL 模式：本地目录路径
  • ☁️ S3/GCS 模式：Bucket 路径（如 s3://my-bucket/ducklake/）
⚠️ 注意事项：
• 需确保有足够的磁盘空间
• 需要读写权限
• 建议使用 SSD 以获得更好性能

📊 实时增量物化视图（6个参数）

实时增量物化视图是什么？
传统模式：全量同步完成 → 执行一次 SQL 查询 → 输出结果（静态快照）
物化视图模式：全量同步 → CDC 实时更新 → 自动刷新查询结果（动态维护）

1️⃣ wideTablePrimaryKey ⭐ 核心必填

类型: String
默认值: null（必填）
示例: "order_id", "user_id"
@EqField: ✅

功能描述：

  • 🔑 定义宽表的主键字段名
  • 🎯 用于 CDC 事件的 UPDATE/DELETE 匹配
  • ⚡ 影响 WideTableIncrementalUpdater 的 upsert 性能
📌 为什么必填？
物化视图需要知道如何识别唯一记录，以便在 CDC 阶段正确更新或删除。

2️⃣ outputChangelogEnabled

类型: Boolean
默认值: false
@EqField: ✅

功能描述：

  • 📤 是否输出宽表的 CDC 变更日志
  • 🔄 true: 每个 INSERT/UPDATE/DELETE 都生成事件
  • 📦 false: 仅在全量完成后输出最终结果
💡 使用场景：
• ✅ true: 下游需要实时感知变化（如实时报表、缓存更新）
• ❌ false: 下游只需要最终结果（如批量导出、离线分析）

3️⃣ mainTableName

类型: String
默认值: null
示例: "orders", "users"
@EqField: ✅

功能描述：

  • 📋 定义物化视图的主表名称
  • 🔗 主表是 JOIN 操作的核心表（FROM 子句的第一个表）
  • 📊 用于确定 CDC 事件的来源优先级

4️⃣ mainTablePrimaryKey

类型: String
默认值: null
示例: "id", "pk"
@EqField: ✅

功能描述：

  • 🔑 主表的主键字段名
  • 🎯 与 wideTablePrimaryKey 配合使用
  • ⚠️ 通常与 wideTablePrimaryKey 相同或相关联

5️⃣ fromTables (List<FromTableConfig>) 📋 复杂对象

类型: List<FromTableConfig>
默认值: new ArrayList<>()
@EqField: ✅

功能描述：

  • 📚 定义从表列表（JOIN 的其他表）
  • 🏗️ 每个 FromTableConfig 包含：
    • tableName: 从表名称
    • primaryKey: 从表主键字段
  • 🔗 用于构建多表 JOIN 关系
📌 示例配置：
[
  { tableName: "order_items", primaryKey: "item_id" },
  { tableName: "customers",   primaryKey: "customer_id" }
]

6️⃣ customJoinQueries (Map<String, String>) 🔧 高级定制

类型: Map<String, String>
默认值: new HashMap<>()
@EqField: ✅

功能描述：

  • 🎨 自定义每个从表的 JOIN 查询模板
  • 🔧 当自动生成的 SQL 不满足需求时使用
  • 📝 支持 ${pkValues} 占位符（动态替换为主键值）
📌 示例配置：
{
  "order_items": 
    "SELECT * FROM order_items WHERE order_id IN (${pkValues})",
  "customers":
    "SELECT * FROM customers WHERE id IN (${pkValues})"
}

❓ executeQueryOnFullSyncComplete - 详细解释

📖 功能定义

控制何时执行 SQL 查询并发射结果到下游。这是一个时序控制开关，决定查询执行的触发时机。

源码逻辑（第 351 行）： 
if (executeQueryOnFullSyncComplete && !queryExecuted) {
    executeAndEmitQueryResults();
    queryExecuted = true;
}

🎯 触发条件

  1. 所有前置表都已完成全量同步
  2. 所有前置表都进入 CDC（增量同步）阶段
  3. syncStageTracker 触发 onAllTablesCdcCallback
  4. 调用 handleAllTablesCdcTransition()

✅ 设为 true（默认行为）

执行流程：
  1. 全量同步阶段：接收数据 → 写入 DuckDB 内存表
  2. 全量完成信号：所有表切换到 CDC
  3. 🔄 立即执行 SQL 查询
  4. 将查询结果作为 TapdataEvent 发射到下游
  5. CDC 阶段：继续接收增量数据（可选启用物化视图）

适用场景：

  • ✅ 批量数据处理任务
  • ✅ 全量+增量混合模式
  • ✅ 需要在 CDC 开始前获得完整快照

❌ 设为 false

执行流程：
  1. 全量同步阶段：接收数据 → 写入 DuckDB
  2. 全量完成信号：所有表切换到 CDC
  3. ⏭️ 跳过查询执行
  4. CDC 阶段：仅通过物化视图机制输出变更
  5. 下游不会收到全量快照结果

适用场景：

  • ✅ 纯 CDC 实时同步任务
  • ✅ 已有历史数据，只需增量更新
  • ✅ 配合物化视图的 outputChangelogEnabled=true

🔄 与物化视图的关系

executeQueryOn... outputChangelog... 实际效果
true false 📸 快照模式: 输出一次完整结果，CDC 阶段静默
true true 🔄 混合模式: 先输出快照，再持续输出 CDC 变更
false true 📡 纯流式模式: 无快照，仅输出 CDC 变更流
false false ⚠️ 异常模式: 不推荐！无任何输出
